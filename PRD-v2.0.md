## 1. 产品概述

### 1.1 产品定位

> **"首个专为孕晚期（28-40周）设计的倒计时 + 任务助手"**

**核心价值：**
- 不是又一个全孕期管理 App（宝宝树/妈妈网太臃肿）
- 专注孕晚期最后 3 个月的核心需求
- 倒计时带来期待感 + 任务管理降低焦虑感

### 1.2 目标用户

**主要用户画像：**
- 孕周：28-40 周（孕晚期）
- 年龄：25-35 岁
- 特征：
  - 第一次怀孕，对分娩有焦虑
  - 想做好准备（待产包、拉玛泽），但不知从何开始
  - 每天期待与宝宝见面
  - 不喜欢臃肿的 App，希望简洁高效

**真实用户案例（Juanjuan）：**
- 孕 37 周+3 天
- 想做凯格尔运动（产后恢复）
- 想学拉玛泽呼吸法（减轻分娩疼痛）
- 需要准备待产包（但不知道要买什么）
- 每天期待预产期到来

### 1.3 核心功能

**4 个 Tab：**

| Tab | 核心功能 | 用户价值 |
|-----|----------|----------|
| 🏠 首页 | 倒计时 + 快速任务入口 | 每日打开的理由 |
| ✅ 任务 | 凯格尔 + 拉玛泽 + 胎动 | 降低分娩焦虑 |
| 🎒 待产包 | Checklist 管理 | 不漏买任何东西 |
| 📖 知识 | 分娩知识卡片 | 科学备产 |

### 1.4 市场验证

**蓝海市场：**
- "待产" 搜索：8 个结果（市场几乎空白）
- "分娩准备" 搜索：4 个结果，全部 0 评论
- "拉玛泽" 搜索：5 个结果，竟然全是游戏 😂
- "产前准备" 搜索：2 个结果，0 评论

**竞品问题：**
- 宝宝树/妈妈网：功能太多（孕期+育儿+社区+电商），臃肿
- 柠檬胎动：只做胎动+宫缩，没有待产包/拉玛泽
- G动：只做凯格尔，不针对孕期
- 孕宝/好孕妈：评分低（3.6★ / 3.17★）

**差异化定位：**
> 我们不做全孕期管理，只做孕晚期最后 3 个月的倒计时 + 任务助手。

#### 2.2.3 凯格尔运动详情

**3 个训练级别（收缩:放松 = 1:2）：**

| 级别 | 收缩时长 | 放松时长 | 组数 | 默认周期 |
|------|----------|----------|------|----------|
| ● 初级 | 5 秒 | 10 秒 | 10 次 | 第 1-7 天 |
| ●● 中级 | 7 秒 | 14 秒 | 10 次 | 第 8-21 天 |
| ●●● 高级 | 10 秒 | 20 秒 | 10 次 | 第 22 天起 |

**级别切换逻辑：**
- 按照「首次打开 App 的日期」自动推进级别，无需用户干预
- 第 1 天 → 初级；第 8 天 → 自动切换中级；第 22 天 → 自动切换高级
- 跳过某天不练习，不影响天数计算（按日历日期，不按练习次数）
- 用户可在设置里手动调整级别（不常用，但提供出口）
- **首次安装日期通过 iCloud KeyValueStore 持久化**，用户删除重装后级别不会重置（v2.1 评审确认）

**点击任务卡片 → 进入练习页面：**

```
┌─────────────────────────────────┐
│         凯格尔运动                │
│    ● 初级 · 5s 收缩 / 10s 放松  │
│                                  │
│    [收缩骨盆底肌]                │
│       ●●●●●○○○○○                │
│                                  │
│    还剩 6 组（共 10 组）          │
│                                  │
│    [暂停]   [结束]               │
└─────────────────────────────────┘
```

**练习流程：**
1. 点击"开始练习" → 显示当前级别（初级：5s/10s）
2. **收缩阶段**：震动提示开始，倒计时 5 秒（进度条动画）
3. 倒计时到 0 → 轻震动提示 → 自动切换放松
4. **放松阶段**：倒计时 10 秒（进度条变灰）
5. 倒计时到 0 → 轻震动提示 → 自动进入下一组
6. 完成 10 组 → 强震动 + 完成提示

**完成后：**
- 跳转全屏完成页：
  - 标题："做得真棒"
  - 副标题："今天的凯格尔运动已完成" + "坚持下去，和宝宝见面那天会更从容"
  - ✓ 图标（SVG checkmark）
  - 底部提示："It's the moment you show up for you."
  - 按钮："返回首页"
- 首页任务卡片状态变成 "✓ 已完成"

## 4.5 推送通知

### 4.5.1 功能目标

- **核心目标：** 每日提醒用户完成凯格尔/拉玛泽练习，养成习惯
- **原则：** 必须用户主动同意后才推送，不强制

### 4.5.2 权限请求流程

#### 触发时机

**不在 Onboarding 后立即请求。** 在用户**首次完成一个任务**后触发：
- 完成一次凯格尔运动
- 完成一次拉玛泽练习
- 勾选一个待产包物品

任选其一触发即可，只触发一次。

#### 两步请求流程

**第 1 步：自定义预请求弹窗（我们设计的，漂亮）**

在首次任务完成后弹出，突出开启通知对用户的好处：

**设计稿：**

![通知预请求弹窗](https://github.com/GhostComplex/Blossom/blob/main/qa-screenshots/design-notification-prerequest.png?raw=true)

**弹窗内容（按触发场景区分）：**

**凯格尔/拉玛泽触发时：**
- 标题：「每天提醒你练习」
- 正文：「每天练几分钟，和宝宝见面那天会更从容。」

**待产包触发时：**
- 标题：「别忘了每天的小任务」
- 正文：「我们会在合适的时候提醒你练习和准备。」

**通用按钮（两种场景一样）：**
- 主按钮：「好的，提醒我」→ 触发第 2 步系统弹窗
- 次按钮：「不了，谢谢」→ 关闭弹窗，不再弹出

**第 2 步：iOS 系统权限弹窗（系统强制，无法自定义样式）**

用户点了「好的，提醒我」后，iOS 自动弹出系统权限弹窗。

**用户同意：**
1. 系统弹窗 → 用户点"允许"
2. App 注册每日本地通知

**用户拒绝：**
1. 系统弹窗 → 用户点"不允许"
2. App 不推送，不再弹窗（iOS 限制：系统权限弹窗只弹一次）
3. 不主动引导用户去开启

**用户在预请求弹窗点「不了，谢谢」：**
- 不触发系统弹窗
- 不再弹出预请求弹窗（保留一次机会给系统弹窗）
- 后续如果用户想开启，需自行前往 iOS 系统设置

**用户先同意后在系统设置关闭：**
- App 下次启动时检测权限状态
- 发现权限被关闭 → 停止注册新通知
- 不弹窗提醒，不影响 App 正常使用

### 4.5.3 推送规则

**目的：提醒用户完成当天未完成的任务，不是刷存在感。**

**凯格尔 / 拉玛泽提醒：**
- **时间：** 每天 17:00
- **逻辑：**
  - 凯格尔未完成 + 拉玛泽未完成 → 推 1 条合并提醒
  - 仅凯格尔未完成 → 推凯格尔提醒
  - 仅拉玛泽未完成 → 推拉玛泽提醒
  - 都已完成 → 不推送
- **取消机制：** 每次 App 打开时注册当天 17:00 通知；用户在 17:00 前完成任务 → 完成瞬间立刻取消当天通知

**待产包提醒：**
- **仅在用户超过 7 天没打开 App 时推送**
- 不按每日推送，按「回归提醒」逻辑
- 实现方式：App 启动时记录最后打开时间，注册 7 天后的一条通知；每次打开 App 时重置这个 7 天计时器

**预产期过后：** 停止所有推送

### 4.5.4 推送文案

| 场景 | 文案 |
|------|------|
| 凯格尔 + 拉玛泽都未完成 | 今天的凯格尔和拉玛泽还没做哦，花几分钟练一下吧 |
| 仅凯格尔未完成 | 今天的凯格尔运动还没做，3 分钟就搞定 |
| 仅拉玛泽未完成 | 今天的拉玛泽呼吸还没练，为分娩做准备吧 |
| 7 天未打开 App | 好久没来了，看看待产包还有什么没准备的？ |

### 4.5.5 技术实现要点

- **框架：** iOS UserNotifications（纯本地推送，不需要后端/APNs）
- **权限请求：** `UNUserNotificationCenter.requestAuthorization(options: [.alert, .sound])`
- **注册通知：** `UNCalendarNotificationTrigger`，每日 17:00 触发
- **文案选择：** 根据当天任务完成状态选择对应文案（见 §4.5.4），不是随机
- **预注册策略：** 每天 App 打开时，预注册**第二天** 17:00 的通知（默认使用「都未完成」的合并文案）；第二天用户打开 App 后，根据实际任务完成状态调整或取消当天通知
- **冷启动兜底：** 如果用户当天完全没打开 App，前一天预注册的通知仍在 17:00 触发（使用合并文案）
- **取消机制：** 用户在 17:00 前完成所有任务 → 完成瞬间取消当天通知
- **通知上限：** iOS 最多 64 条待发通知，孕晚期最多 90 天，够用
- **App 被杀掉：** 本地通知由 iOS 系统调度，App 不在前台也能触发
- **卸载：** 自动清除所有已注册通知

### 4.5.6 不做的事（MVP 范围外）

- ❌ 不加设置页（用户不能在 App 内修改通知时间）
- ❌ 不做动态文案（X 天倒计时）——本地通知内容注册时固定
- ❌ 不做通知点击跳转到具体页面（点击通知只打开 App 首页）
- ❌ 不做二次权限引导（用户拒绝后不再提醒）

以上可在 v1.1 中按需添加。

### 4.5.7 验收标准

| # | 验收项 |
|---|--------|
| 1 | 首次完成凯格尔/拉玛泽/勾选待产包后弹出自定义预请求弹窗 |
| 2 | 预请求弹窗文案突出好处，有「好的，提醒我」和「不了，谢谢」两个按钮 |
| 3 | 点「好的，提醒我」→ 弹出 iOS 系统权限弹窗 |
| 4 | 点「不了，谢谢」→ 关闭弹窗，不再弹出 |
| 5 | 系统权限同意后，17:00 前未完成凯格尔 → 17:00 收到凯格尔提醒 |
| 6 | 17:00 前未完成拉玛泽 → 17:00 收到拉玛泽提醒 |
| 7 | 凯格尔 + 拉玛泽都未完成 → 收到 1 条合并提醒（不是 2 条） |
| 8 | 17:00 前都完成了 → 不收到通知 |
| 9 | 拒绝权限后不收到任何通知 |
| 10 | 预产期过后不再收到通知 |

```swift
func calculateDaysLeft(dueDate: Date) -> Int {
    let calendar = Calendar.current
    let today = calendar.startOfDay(for: Date())
    let due = calendar.startOfDay(for: dueDate)
    let components = calendar.dateComponents([.day], from: today, to: due)
    return components.day ?? 0
}
```

**孕周计算：**
```swift
func calculatePregnancyWeek(dueDate: Date) -> (week: Int, day: Int) {
    let calendar = Calendar.current
    let today = Date()
    let conceptionDate = calendar.date(byAdding: .day, value: -280, to: dueDate)!
    let days = calendar.dateComponents([.day], from: conceptionDate, to: today).day ?? 0
    return (week: days / 7, day: days % 7)
}
```

**凯格尔计时器：**
```swift
class KegelTimer: ObservableObject {
    @Published var currentPhase: Phase = .contract
    @Published var timeLeft: Int = 10
    @Published var setsCompleted: Int = 0
    
    enum Phase {
        case contract, relax
    }
    
    func start() {
        Timer.publish(every: 1, on: .main, in: .common)
            .autoconnect()
            .sink { _ in
                self.tick()
            }
    }
    
    func tick() {
        if timeLeft > 0 {
            timeLeft -= 1
        } else {
            switchPhase()
        }
    }
    
    func switchPhase() {
        if currentPhase == .contract {
            currentPhase = .relax
            timeLeft = 10
            HapticFeedback.medium()
        } else {
            setsCompleted += 1
            if setsCompleted < 10 {
                currentPhase = .contract
                timeLeft = 10
                HapticFeedback.medium()
            } else {
                complete()
            }
        }
    }
}
```
