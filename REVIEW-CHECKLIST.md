## ⚠️ 可能被 Challenge 的点

### 1. 设计相关

**可能的质疑：**

**Q1: "为什么用 HTML 做设计稿，不用 Figma？"**
- **回答：** HTML 是可交互原型，开发可以直接复制 CSS 代码，比静态设计稿更高效。所有颜色、间距、圆角都已经用 CSS Variables 定义好，开发可以直接在 SwiftUI 中复用。

**Q2: "玫瑰金 #C9A084 对比度不够（3.8:1），WCAG AA 要求 4.5:1"**
- **回答：** 玫瑰金只用于装饰元素（图标、边框、按钮背景），不用于小文字。所有重要文字都是 neutral-900 / neutral-700，对比度 9.2:1+，符合 WCAG AAA。

**Q3: "毛玻璃效果（backdrop-filter）在旧设备上性能差"**
- **回答：** 
  - 我们只在 4 个主卡片上用毛玻璃，数量可控
  - iOS 15+ 原生支持，性能优化好
  - 如果真有性能问题，可以降级到纯白背景（已在 CSS 中设置 fallback）

**Q4: "为什么不用 SF Symbols 图标，要自己画 SVG？"**
- **回答：** 可以改用 SF Symbols！HTML 原型里用 SVG 是为了快速预览，开发时全部换成 SF Symbols 更规范。

### 2. 功能相关

**可能的质疑：**

**Q5: "凯格尔计时器震动反馈会很耗电"**
- **回答：** 
  - 每次震动只有 0.05 秒（轻量震动）
  - 一天最多 10 分钟练习，震动 20 次
  - 对电量影响可忽略（参考 Apple Watch 每小时震动数十次）

**Q6: "拉玛泽跟练动画怎么实现？圆圈放大缩小？"**
- **回答：** 
  - SwiftUI `Animation(.easeInOut(duration: 4))` 实现
  - 用 `.scaleEffect()` 控制 0.8 → 1.2 → 0.8
  - 已验证可行（类似 Calm 冥想呼吸动画）

**Q7: "待产包 50 项清单，用户会不会嫌麻烦？"**
- **回答：** 
  - 分类折叠设计，默认只显示分类标题
  - 用户可以"隐藏已完成"，只看未准备的
  - 50 项是参考医院待产包清单，都是必需品

**Q8: "知识文章内容从哪来？需要专业审核吗？"**
- **回答：** 
  - MVP 阶段：从权威来源（WHO、国家卫健委、三甲医院）整理
  - 长期：找产科医生审核
  - 免责声明："内容仅供参考，具体请咨询医生"

### 3. 技术相关

**可能的质疑：**

**Q9: "SwiftData 稳定吗？为什么不用 Core Data？"**
- **回答：** 
  - SwiftData 是 iOS 17+ 新框架，更现代
  - 我们的数据结构简单（用户+任务+待产包），SwiftData 够用
  - 如果团队更熟悉 Core Data，可以换

**Q10: "本地存储，用户换手机怎么办？"**
- **回答：** 
  - MVP 阶段：纯本地，不支持同步
  - v1.2（2 个月后）：加 iCloud 同步
  - 用户痛点不大（孕晚期只用 3 个月，换手机概率低）

**Q11: "推送通知时间写死了（9:00），用户想自定义怎么办？"**
- **回答：** 
  - MVP 阶段：固定 9:00
  - 收集用户反馈后再加"自定义提醒时间"
  - 避免过早优化

**Q12: "倒计时为什么不用服务器时间？本地时间可能不准"**
- **回答：** 
  - 纯本地 App，无需服务器
  - 用户修改手机时间的场景极少
  - 如果真的不准，用户可以在首页点击卡片修改预产期

### 4. 产品相关

**可能的质疑：**

**Q13: "买断制 ¥18，收入太低，不如做订阅"**
- **回答：** 
  - 目标用户使用周期短（3 个月），订阅不合适
  - ¥18 心理负担小，转化率更高
  - 保守估计：1000 月活 × 30% 转化 = ¥5400/月
  - 如果真的跑起来，可以涨到 ¥25

**Q14: "App Store 搜索'待产'只有 8 个结果，是不是伪需求？"**
- **回答：** 
  - 不是伪需求，是蓝海市场
  - 用户真实痛点：Juanjuan 就是目标用户
  - 现有 App（宝宝树/妈妈网）太臃肿，用户需要简洁工具

**Q15: "功能太少，用户留存率会不会很低？"**
- **回答：** 
  - 目标用户明确（孕晚期 28-40 周）
  - 核心功能刚需（倒计时+凯格尔+拉玛泽+待产包）
  - 预估留存率：Day 7 = 70%+（参考 Flo 类健康 App）

**Q16: "为什么不做社区？用户可以分享经验"**
- **回答：** 
  - 社区运营成本高（审核/管理/内容质量）
  - MVP 阶段专注工具属性
  - 如果用户真的有需求，可以 v2.0 加入
