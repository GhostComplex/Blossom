## 1. 背景

当前拉玛泽有 3 个入口：跟练模式、学习模式、知识卡片。

**问题：**
- 学习模式内容太干巴巴（只有"吸气 4 秒呼气 6 秒"），用户看不懂拉玛泽是什么
- 知识卡片没有定义点击行为（PRD 遗漏）
- 学习模式和知识卡片功能重叠——都是"看文字了解拉玛泽"

## 2. 目标

合并学习模式 + 知识卡片为一个"了解拉玛泽"入口，用人话重写 6 阶段内容，让用户真正看懂拉玛泽呼吸法。

## 4. 六阶段教程内容

### 第 1 阶段：清洁呼吸
- 什么时候用：每次宫缩开始和结束时
- 为什么：帮你从紧张切换到放松，像一个呼吸的"开关"
- 怎么做：鼻子慢慢吸气 4 秒 → 嘴巴慢慢呼气 6 秒，重复 6 次

### 第 2 阶段：胸式呼吸
- 什么时候用：宫缩初期，开指 0-3cm，间隔 5-20 分钟
- 为什么：宫缩还不太强时，用缓慢的胸部呼吸保持平静
- 怎么做：鼻子吸气 4 秒 → 嘴巴呼气 4 秒，节奏平稳

### 第 3 阶段：节律呼吸
- 什么时候用：活跃期，开指 3-8cm，宫缩变强变频繁
- 为什么：随着宫缩加强，呼吸跟着加快，帮你"骑"在宫缩上面
- 怎么做：吸气 3 秒 → 呼气 3 秒，比第 2 阶段稍快

### 第 4 阶段：喘息呼吸
- 什么时候用：过渡期，开指 7-10cm，宫缩最强的时候
- 为什么：这是最难熬的阶段，浅快的呼吸帮你"度过"每一波宫缩
- 怎么做：嘴巴浅快呼吸，像说"嘻-嘻-呼"，每组 1 秒吸 1 秒呼

### 第 5 阶段：吹气呼吸
- 什么时候用：想用力但医生说还不能用力的时候
- 为什么：吹气能抑制用力的冲动，帮你等到正确的时机
- 怎么做：像吹蜡烛一样用力吹气，转移注意力

### 第 6 阶段：用力呼吸
- 什么时候用：全开（10cm），医生说可以用力了
- 为什么：配合用力的呼吸让每一次推更有效
- 怎么做：深吸一口气 → 憋住 → 跟着宫缩用力推

## 5. 页面开头简介

> 拉玛泽呼吸法是一种帮助你在分娩时保持平静的呼吸技巧。通过不同阶段的呼吸方式，配合宫缩节奏，让你在最需要的时候找到掌控感。

## 6. 交互细节

### 6.1 导航流程
```
Tasks Tab
  └→ 拉玛泽卡片
      └→ Mode Selection（跟练 / 了解拉玛泽）
          ├→ 跟练模式（不变）
          └→ 了解拉玛泽
              ├→ 阅读 6 阶段内容
              └→ "试试这个阶段" → Prep 页 → Timer → 完成页
                                                      └→ "继续了解" → 回到了解页
```

### 6.2 返回逻辑
| 当前页面 | 返回去哪 |
|---------|---------|
| 了解拉玛泽 | Mode Selection 页 |
| Prep 页（从了解进入） | 了解拉玛泽 |
| Timer（从了解进入） | 了解拉玛泽 |
| 完成页（从了解进入） | 了解拉玛泽（按钮文案"继续了解"） |

### 6.3 "试试这个阶段"按钮
- 点击 → 直接进入该阶段的 prep 页（跳过 mode selection 和阶段选择）
- 复用现有 prep → timer → 完成 流程
- 区别：完成页按钮文案从"返回首页"改为"继续了解"，点击后回到了解页面

### 6.4 卡片状态
- 6 个阶段卡片全部展开，不做折叠/展开交互
- ScrollView 垂直滚动

## 9. 开发注意事项

1. Mode selection 页删除"学习模式"和"知识卡片"入口，保留"跟练模式"，新增"了解拉玛泽"
2. 新建 `LamazeLearningView`（了解拉玛泽页面），内容硬编码在 Swift 里（不需要后端）
3. "试试这个阶段"按钮需要传递 stage 参数给 prep 页
4. 完成页需要根据来源（跟练 vs 了解）决定按钮文案和返回目标
5. 导航栈：了解页面 push prep → push timer → push 完成 → pop 到了解页面
